## SECTION 7 — UI & UX COMPLETENESS

7.1 **Loading states**: Every async operation (AsyncStorage read, mock AI response, auto-generate plan, party planner generation) must show a loading spinner or skeleton. No operation should leave the UI frozen with no feedback.

7.2 **Empty states**: Every list in the app (pantry, saved recipes, cooked recipes, meal plan, social feed, notifications) must have a designed empty state with a helpful message and a CTA to add content. No blank white screens.

7.3 **Error states**: Network errors (for any future real API calls), AsyncStorage failures, and invalid data must show user-friendly error messages with a retry option. No raw error objects rendered to screen.

7.4 **Form validation**: Every input field across the app (add ingredient, onboarding preferences, party planner wizard) must validate on submit. Required fields show errors when empty. Numeric fields reject non-numeric input. Date fields reject invalid dates.

7.5 **Keyboard handling**: Every screen with a text input must handle the keyboard correctly — KeyboardAvoidingView, ScrollView with keyboardShouldPersistTaps, or equivalent. No input should be hidden behind the keyboard.

7.6 **Button states**: Every button must have a disabled state when its action is not available (e.g. "Next" on a form step with empty required fields). Disabled buttons must look visually distinct and not respond to taps.

7.7 **Double-tap / rapid tap protection**: All buttons that trigger state changes (swipe, like, save, add item, send message) must be debounced or guarded to prevent double-firing on rapid taps. Use a ref-based guard or `disabled` prop during async operations.

7.8 **Haptic feedback**: Confirm `expo-haptics` fires on: successful swipe right, successful swipe left, save (swipe up), like, achievement unlocked, level up, and completing a recipe. These should feel tactile and intentional.

7.9 **Brand colors**: Confirm `constants/colors.ts` is imported and used consistently. No hardcoded hex color values scattered in component styles — all colors must reference the constants. Audit 5 random components for compliance.

7.10 **Dark mode**: Confirm the app respects the device color scheme. All screens must be readable in both light and dark mode. No white-on-white or black-on-black text. Test the color constants for both themes.

7.11 **Font sizes & accessibility**: No text smaller than 12sp. Interactive tap targets no smaller than 44×44pt. Verify on a small screen (375pt width).

7.12 **Image loading**: All recipe and post images must show a placeholder/skeleton while loading and a fallback if the image fails to load. No broken image icons visible to users.

7.13 **Scroll performance**: Long lists (recipes, pantry items, social feed) must use `FlatList` or `FlashList`, NOT `ScrollView` with `.map()`. Confirm this in every screen with a list. Fix any that use map-in-ScrollView.
