## SECTION 1 — ARCHITECTURE & ORIENTATION

### Why Supabase
- Built-in Auth (Google OAuth + email/password), token refresh, and session management
- `@supabase/supabase-js` works identically in Expo and Node.js — one library for both sides of the monorepo
- Hosted Postgres with Row Level Security
- Free-tier sufficient for this stage

### What changes vs what stays

**Changes:**
- `app/index.tsx` — auth-aware routing replaces the AsyncStorage flag check
- `app/welcome.tsx` — gains Google Sign-In + email/password form
- `app/onboarding.tsx` — triggered only after auth, for new users only
- `context/AppContext.tsx` — gains session, profile, and auth actions
- `artifacts/api-server/` — gains Supabase client for token verification

**Stays the same (do not touch):**
- All tab screens, swipe cards, cook mode, AI Chef, party planner
- `app/(tabs)/_layout.tsx` — contains iOS 26 NativeTabs logic, must be preserved

### Auth flow — NEW user:
```
App open → index.tsx: no session → /welcome
→ Google Sign-In → Supabase PKCE exchange → session created
→ onAuthStateChange fires → fetchProfile → null (no row yet)
→ index.tsx: authUser set, profile null → /onboarding
→ User completes onboarding → completeOnboarding() upserts profile row
→ profile.onboardingComplete = true → router.replace('/(tabs)')
```

### Auth flow — RETURNING user:
```
App open → index.tsx: valid persisted session found
→ fetchProfile → profile row found → router.replace('/(tabs)')
→ Session auto-refreshes in background — user never re-authenticates
```

### Auth flow — SIGN OUT:
```
Profile tab → Sign Out confirmed → supabase.auth.signOut()
→ onAuthStateChange fires with null session
→ AppContext clears profile → index.tsx detects no authUser → /welcome
```

### 2.3 — Enable Row Level Security (CRITICAL — never skip)

```sql
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own profile"
  ON profiles FOR SELECT
  USING (auth.uid() = id);

CREATE POLICY "Users can insert own profile"
  ON profiles FOR INSERT
  WITH CHECK (auth.uid() = id);

CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE
  USING (auth.uid() = id)
  WITH CHECK (auth.uid() = id);
```

Without RLS any authenticated user can read every other user's data. This is non-negotiable.

### 6.2 — Auth implementation inside AppProvider

⚠️ **Critical ordering rule: `fetchProfile` must be declared BEFORE the `useEffect` that calls it.** JavaScript `const` arrow functions are not hoisted — calling them before declaration is a runtime error.

```typescript
import { supabase } from '../lib/supabase';
import { mapProfileFromDB, UserProfile, PantryItem } from '../types/profile';

// --- State ---
const [session, setSession] = useState<Session | null>(null);
const [authUser, setAuthUser] = useState<User | null>(null);
const [profile, setProfile] = useState<UserProfile | null>(null);
const [isLoadingAuth, setIsLoadingAuth] = useState(true);

// --- fetchProfile: declared FIRST, before any useEffect that calls it ---
const fetchProfile = useCallback(async (userId: string) => {
  try {
    const { data, error } = await supabase
      .from('profiles')
      .select('*')
      .eq('id', userId)
      .single();

    if (error) {
      if (error.code === 'PGRST116') {
        // Row not found = brand new user, profile not yet created
        setProfile(null);
      } else {
        console.error('[AppContext] fetchProfile error:', error.message);
        setProfile(null);
      }
    } else {
      setProfile(mapProfileFromDB(data));
    }
  } finally {
    setIsLoadingAuth(false);
  }
}, []); // no deps — supabase client is stable

// --- Session init: runs once on mount ---
// ⚠️ Only use onAuthStateChange — do NOT also call getSession() separately.
// Supabase fires onAuthStateChange with the restored session on mount (INITIAL_SESSION event),
// so getSession() is redundant and causes a double fetchProfile race condition.
useEffect(() => {
  const { data: { subscription } } = supabase.auth.onAuthStateChange(
    async (event, session) => {
      setSession(session);
      setAuthUser(session?.user ?? null);

      if (session?.user) {
        // SIGNED_IN, TOKEN_REFRESHED, USER_UPDATED, INITIAL_SESSION with valid session
        await fetchProfile(session.user.id);
      } else {
        // SIGNED_OUT, INITIAL_SESSION with no session
        setProfile(null);
        setIsLoadingAuth(false);
      }
    }
  );

  return () => subscription.unsubscribe();
}, [fetchProfile]);

// --- signOut: let onAuthStateChange handle state clearing ---
// Do NOT manually call setSession/setAuthUser/setProfile here.
// onAuthStateChange fires with session=null after signOut(), which clears everything.
const signOut = async () => {
  await supabase.auth.signOut();
  // onAuthStateChange (SIGNED_OUT event) handles the state reset
};

// --- updateProfile: optimistic update FIRST, then persist, roll back on error ---
const updateProfile = async (updates: Partial<UserProfile>) => {
  if (!authUser) return;
  if (!profile) return;

  // 1. Optimistic update — UI responds instantly without waiting for network
  const previousProfile = profile;
  setProfile(prev => prev ? { ...prev, ...updates } : prev);

  // 2. Build snake_case DB payload
  const dbUpdates: Record<string, unknown> = {};
  if (updates.displayName !== undefined) dbUpdates.display_name = updates.displayName;
  if (updates.avatarUrl !== undefined) dbUpdates.avatar_url = updates.avatarUrl;
  if (updates.dietaryPreferences !== undefined) dbUpdates.dietary_preferences = updates.dietaryPreferences;
  if (updates.pantryItems !== undefined) dbUpdates.pantry_items = updates.pantryItems;
  if (updates.cookedRecipeIds !== undefined) dbUpdates.cooked_recipe_ids = updates.cookedRecipeIds;
  if (updates.savedRecipeIds !== undefined) dbUpdates.saved_recipe_ids = updates.savedRecipeIds;
  if (updates.xp !== undefined) dbUpdates.xp = updates.xp;
  if (updates.streak !== undefined) dbUpdates.streak = updates.streak;
  if (updates.lastCookedDate !== undefined) dbUpdates.last_cooked_date = updates.lastCookedDate;
  if (updates.recipesSwipedCount !== undefined) dbUpdates.recipes_swiped_count = updates.recipesSwipedCount;
  if (updates.partyPlansCreated !== undefined) dbUpdates.party_plans_created = updates.partyPlansCreated;
  if (updates.totalLikesGiven !== undefined) dbUpdates.total_likes_given = updates.totalLikesGiven;
  if (updates.onboardingComplete !== undefined) dbUpdates.onboarding_complete = updates.onboardingComplete;

  // 3. Persist to Supabase
  const { error } = await supabase
    .from('profiles')
    .update(dbUpdates)
    .eq('id', authUser.id);

  if (error) {
    // 4. Roll back optimistic update — restore previous state
    console.error('[AppContext] updateProfile failed, rolling back:', error.message);
    setProfile(previousProfile);
    // Surface error to caller so the UI can show a message
    throw new Error('Failed to save changes. Please check your connection and try again.');
  }
};

// --- completeOnboarding: creates the profile row for new users ---
// Uses upsert (not insert) because:
//   - insert fails if the row somehow already exists (e.g. user tapped twice before disabled state kicked in)
//   - upsert = INSERT ... ON CONFLICT DO UPDATE, which is idempotent and safe
// The edited displayName from onboarding Step 1 is passed as a parameter — NOT read from user_metadata.
const completeOnboarding = async (prefs: {
  displayName: string;
  dietaryPreferences: string[];
  pantryItems: PantryItem[];
}) => {
  if (!authUser) throw new Error('Not authenticated');

  const { error } = await supabase
    .from('profiles')
    .upsert({
      id: authUser.id,
      email: authUser.email ?? '',
      display_name: prefs.displayName,                            // from user's Step 1 input
      avatar_url: authUser.user_metadata?.avatar_url ?? null,    // from Google, null for email users
      dietary_preferences: prefs.dietaryPreferences,
      pantry_items: prefs.pantryItems,
      onboarding_complete: true,
    }, {
      onConflict: 'id',   // explicit conflict target — safer than letting Supabase infer
    });

  if (error) {
    console.error('[AppContext] completeOnboarding error:', error.message);
    throw error;
  }

  // Fetch fresh profile from DB — do not optimistically set,
  // because we want server-generated defaults (xp=0, streak=0, etc.) to load correctly
  await fetchProfile(authUser.id);
};
```

Expose all new values and functions in the context value object.

## SECTION 8 — UPDATE app/welcome.tsx (SIGN-IN SCREEN)

### 8.1 — Google Sign-In (PKCE-correct implementation)

⚠️ **Supabase Auth v2 uses PKCE by default.** This means after Google sign-in, Supabase sends a `?code=...` authorization code to the redirect URL — NOT a `#access_token=...` fragment. Do NOT use `AuthSession.parseRedirectParams` to extract tokens. Do NOT call `supabase.auth.setSession()` manually. Instead, use the pattern below where the deep link callback (Section 13) handles the PKCE code exchange automatically.

```typescript
import * as WebBrowser from 'expo-web-browser';
import * as Linking from 'expo-linking';
import { supabase } from '../lib/supabase';
import { Alert, ActivityIndicator } from 'react-native';

// Required: tells expo-web-browser to complete the session when the browser closes on Android
WebBrowser.maybeCompleteAuthSession();

const [isGoogleLoading, setIsGoogleLoading] = useState(false);

const handleGoogleSignIn = async () => {
  setIsGoogleLoading(true);
  try {
    // Build the redirect URI using the custom scheme defined in app.json
    // Supabase will redirect to: pantryswipe://auth/callback?code=...
    const redirectUri = Linking.createURL('auth/callback');

    const { data, error } = await supabase.auth.signInWithOAuth({
      provider: 'google',
      options: {
        redirectTo: redirectUri,
        skipBrowserRedirect: true,   // we open the browser manually below
        queryParams: {
          access_type: 'offline',
          prompt: 'consent',         // ensures Google returns a refresh token
        },
      },
    });

    if (error) throw error;
    if (!data.url) throw new Error('No OAuth URL returned from Supabase');

    // Open Google sign-in in the system browser
    const result = await WebBrowser.openAuthSessionAsync(data.url, redirectUri);

    if (result.type === 'cancel') {
      // User closed the browser without signing in — not an error
      return;
    }

    if (result.type === 'success') {
      // The PKCE code exchange is handled by the deep link handler in _layout.tsx (Section 13).
      // onAuthStateChange in AppContext will fire once the exchange completes.
      // No manual token extraction or setSession call needed here.
    }

  } catch (err) {
    const message = err instanceof Error ? err.message : 'Something went wrong.';
    Alert.alert('Sign-in failed', message);
  } finally {
    setIsGoogleLoading(false);
  }
};
```

### 8.2 — Email/Password fallback

```typescript
const [email, setEmail] = useState('');
const [password, setPassword] = useState('');
const [showPassword, setShowPassword] = useState(false);
const [isNewAccount, setIsNewAccount] = useState(false);

const handleEmailAuth = async () => {
  if (!email.trim() || !password.trim()) {
    Alert.alert('Required', 'Please enter your email and password.');
    return;
  }
  if (password.length < 6) {
    Alert.alert('Password too short', 'Password must be at least 6 characters.');
    return;
  }

  setIsEmailLoading(true);
  try {
    if (isNewAccount) {
      const { error } = await supabase.auth.signUp({ email: email.trim(), password });
      if (error) throw error;
      Alert.alert(
        'Check your email',
        `We sent a confirmation link to ${email.trim()}. Click it, then sign in here.`,
      );
    } else {
      const { error } = await supabase.auth.signInWithPassword({
        email: email.trim(),
        password,
      });
      if (error) {
        // Map Supabase error messages to user-friendly text
        if (error.message.toLowerCase().includes('invalid login credentials')) {
          throw new Error('Incorrect email or password. Please try again.');
        }
        if (error.message.toLowerCase().includes('email not confirmed')) {
          throw new Error('Please confirm your email first. Check your inbox.');
        }
        throw error;
      }
      // onAuthStateChange fires → AppContext updates → index.tsx routes to correct screen
    }
  } catch (err) {
    const message = err instanceof Error ? err.message : 'Something went wrong.';
    Alert.alert('Error', message);
  } finally {
    setIsEmailLoading(false);
  }
};
```

## SECTION 13 — HANDLE OAUTH DEEP LINK CALLBACK (PKCE EXCHANGE)

This section completes the Google Sign-In flow. After the user signs in via the browser, Google redirects to `pantryswipe://auth/callback?code=XXXX`. The app must catch this URL and give the code to Supabase to exchange for a session.

In `artifacts/pantryswipe/app/_layout.tsx` (root layout — NOT a tab layout), add:

```typescript
import * as Linking from 'expo-linking';
import { useEffect } from 'react';
import { supabase } from '../lib/supabase';

// Inside the root layout component body:
useEffect(() => {
  const handleDeepLink = async (url: string) => {
    // Only process auth callback URLs
    if (!url.includes('auth/callback')) return;

    // Parse the URL to extract the code parameter
    const { queryParams } = Linking.parse(url);
    const code = queryParams?.code;

    if (typeof code === 'string') {
      // Exchange the PKCE authorization code for a session
      // This is the correct Supabase v2 PKCE completion method
      const { error } = await supabase.auth.exchangeCodeForSession(code);
      if (error) {
        console.error('[Auth] PKCE exchange failed:', error.message);
        // User will remain on the welcome screen; they can try again
      }
      // On success: onAuthStateChange fires with the new session
      // AppContext updates → index.tsx routes to onboarding or tabs
    }
  };

  // Cold start: app was closed when the deep link arrived
  Linking.getInitialURL().then(url => {
    if (url) handleDeepLink(url);
  });

  // Warm start: app was in background
  const subscription = Linking.addEventListener('url', ({ url }) => handleDeepLink(url));

  return () => subscription.remove();
}, []); // runs once on mount
```

**Why `exchangeCodeForSession` and not `getSessionFromUrl`:**
- `getSessionFromUrl` reads from `window.location` — it is web-only and does nothing in React Native
- `exchangeCodeForSession(code)` is the correct Supabase v2 React Native PKCE completion method
- It takes the `code` query parameter, sends it to Supabase, and returns a session
- No `as never` cast needed — this is the documented API for this use case

## IMPORTANT CONSTRAINTS

- Supabase **anon key** is safe in the Expo app — RLS enforces data isolation. **Service role key must NEVER appear in the app.**
- Do NOT call `supabase.auth.getSession()` separately from `onAuthStateChange` — the listener handles the initial session on mount (INITIAL_SESSION event). Calling both causes a double-fetchProfile race condition.
- Do NOT call `supabase.auth.setSession()` manually after Google sign-in — `exchangeCodeForSession(code)` handles it.
- Do NOT use `getSessionFromUrl()` — it is web-only and does nothing in React Native.
- Do NOT use `AuthSession.parseRedirectParams` — that is for implicit flow; Supabase v2 uses PKCE.
- Do NOT modify `app/(tabs)/_layout.tsx`.
- Do NOT add guest/anonymous sign-in.
- Do NOT break existing features — auth is a new layer on top of the existing app.
- After ANY `.env` change, restart Metro fully before testing.
